Hard rules

From services/operator_console/design_system/DESIGN_SYSTEM.md. These are checked by the design-system audit helper, not negotiated per page.

01 Do not compose QSS inline in views/ or widgets/. The single composed stylesheet from build_stylesheet() is the only place styles are written.
02 Keep hex literals in tokens.py only. Every other module references the typed Palette dataclass.
03 Add any new QSS #ObjectName selector to design_system.json. The manifest is the source of truth for what selectors exist.
04 Use UiStatusKind for status mapping rather than ad-hoc string categories. Never invent a new status word — pick a bucket.
05 Keep views bound to viewmodels and formatter helpers, not API clients. The shell does not talk to the API; it pushes route and selection state into OperatorStore.
