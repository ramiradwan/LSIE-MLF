Setup launcher

The first-run + repair surface that runs before the Operator Console proper. Built in PySide6 like the rest of the console, but lives in services/desktop_launcher/ with its own object-name family. The styling now sits inside design_system/qss_builder.py as a sibling builder — same palette, intentionally larger radii and bolder type to suit the install moment.

i
Now unified with the design system. Per commit a67e1ee, the launcher imports install_setup_stylesheet() from the same module that styles the console. The audit's §11.4 caveat — "setup styling is adjacent to the console's visual system… not yet unified" — no longer holds. The setup surface and the main console now share tokens.py as a single source of truth.

SetupWindow Centered panel · progress · log · context-dependent action buttons

A single centered SetupPanel on the dark app background. The panel hosts a tall heading, a one-line status, a determinate progress bar, a rolling install log, and one of three action buttons depending on lifecycle state. Minimum window size is 680×460.

SetupActionState Five states · one button visible at a time

The setup window shows exactly one action button at a time. Lifecycle is driven by InstallManager.signals — the UI does not own state. Buttons hide rather than disable; the operator should never wonder whether to click again.

Tokens shared palette · two intentional geometry deviations

Setup uses the same palette as the console — same neutrals, same accent, same status_ok green. It intentionally diverges from console geometry on two points: the panel radius is 14px (vs. console's 6px panel radius) and progress / log use 8px radii. The bigger radii signal "this is a moment, not a tool" — appropriate for first-run and repair, jarring on a dashboard. Buttons stay at the console's 6px.
