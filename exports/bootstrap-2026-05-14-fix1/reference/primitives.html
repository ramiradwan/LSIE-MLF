Primitives

Seven shared primitives carry every screen of the console. Each has a fixed object-name set used by the QSS selector layer; any new selector must register in design_system.json. Together they enforce a single way to show a value, a state, a banner, an empty surface, a heading, and a timeline.

SectionHeader

SectionHeader title + optional subtitle · layout-only · accessible name synced from visible copy

The page or panel heading. Always title-case-ish — short, factual, no marketing voice. The subtitle, when present, gives the operator one sentence of context for the surface they just landed on.

MetricCard

MetricCard title · primary value · optional secondary · optional StatusPill · optional clickable

The most-used readback primitive in the console. A card explains a single value, with a muted label, a primary number, and a brief secondary explanation. Cards are not KPI tiles — they are operator readback. When clickable (e.g. Active Session on Overview), hover ring uses --accent.

StatusPill

StatusPill colored dot + text · driven by UiStatusKind

The console's tightest contract: every state shown to the operator goes through the same pill, keyed from UiStatusKind. The dot carries the hue; the label carries the word. Never rely on color alone — the word always rides with the dot.

AlertBanner

AlertBanner severity-coded · hidden when clear · glyph + message

Page-scoped feedback. Banners never collapse into one global error state — they live on the page that owns the failure. Severity is encoded in both the tinted background and the border. Glyph is a plain character, no decorative iconography.

EmptyStateWidget

EmptyState centered title + explanatory message · shown instead of blanks

Blank surfaces erode trust. When a page or panel has no current data, the empty state explains why — and what action, if any, will populate it. Copy is operator-language: causal, non-speculative, never marketing.

EventTimelineWidget

EventTimeline generic table wrapper · width-band column policies · scroll_to_latest()

A reusable table for live timelines and history (encounters, alerts, probes). Rows are dense (8/12px padding); the timestamp column is monospaced; the latest row autoscrolls into view on Live Session. Selection background uses the accent at low alpha.
