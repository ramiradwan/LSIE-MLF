ActionBar

The console's only persistent write surface. The ActionBar mounts below the stacked content and stays present across every route — so the operator's primary action is always one glance away. Its state, copy, and timing all derive from StimulusUiContext; the bar itself owns no backend logic.

Anatomy

The bar is a single row of four regions: session context on the left, an optional operator note, the countdown when measuring is active, and the submit button paired with a lifecycle pill on the right. A second-line message appears below when there is progress or error copy to relay.

Lifecycle

Six lifecycle states, each with a fixed button label, pill, and behavior. The mapping is non-negotiable: changing copy here would silently change what the operator believes the system is doing. completed intentionally returns to Send Stimulus so the bar is immediately usable for the next encounter.

StimulusActionState Pill Meaning Button label
idle idle Session ready, no stimulus in flight, safe-submit guard satisfied. Send Stimulus
submitting submitting POST in flight to orchestrator. Submit disabled to prevent duplicate dispatch. Sending…
accepted accepted Server acknowledged; awaiting first measuring tick from orchestrator. Accepted
measuring measuring Response window is open; countdown reflects orchestrator-owned timing. Measuring…
completed completed Encounter closed. Bar returns to a sendable state for the next stimulus. Send Stimulus
failed failed Submit rejected or window aborted. Bar surfaces error copy and offers Retry. Retry
