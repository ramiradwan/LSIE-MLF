Operator language

The console is read by one trained operator, for hours, under load. Copy is part of the system: it has to be short, causal, and honest about what the pipeline did. formatters.py centralises this language so a state never gets two different names on two different surfaces.

Principles

01
Readback over abstraction
Tell the operator what the system observed and how it treated it. Don't expose backend labels in raw form, but don't editorialise either.
02
Distinguish absence, invalidity, and failure
"No snapshot yet", "snapshot exists but no variability", "stale snapshot", "null-valid co-modulation", and "failure" are five different sentences. Never collapse them.
03
Causal, not speculative
"Reward held back: gate did not confirm" — yes. "It looks like the model isn't sure" — no. We never speculate about a black-box reason; we report the gate result.
04
Recovering ≠ degraded ≠ error
The console enforces this distinction in copy and color. A subsystem returning to healthy is recovering; one operating at reduced capacity is degraded; one that has failed is error.
05
Short, lower-case, no marketing voice
Status pills are lower-case single words. Banner messages are one sentence. No exclamations. No emoji. The operator is busy.
06
Same word, every surface
UiStatusKind, AlertSeverity, HealthState, StimulusActionState — these are the buckets. Every page draws from the same vocabulary.

The shared vocabulary

Four enum types provide the entire status vocabulary used across the console. Don't introduce a new word — pick the bucket.

UiStatusKind
state.py · drives StatusPill, MetricCard, Health rows
  • okHealthy. Probe passing. Reward counted.
  • warnStale, soft timeout, accepted but pending.
  • badHard failure. Submit failed. Probe error.
  • recoveringReturning to healthy after a fault.
  • degradedOperating at reduced capacity. Not an error.
  • mutedNot configured / not applicable.
AlertSeverity
state.py · drives AlertBanner
  • infoOperator should know, no action required.
  • warnAction may be required soon; nothing is failing yet.
  • criticalAction required now to keep the session usable.
HealthState
state.py · drives Health rollup, subsystem rows
  • okSubsystem fully operational.
  • degradedOperational with reduced capacity.
  • recoveringReturning from a transient fault.
  • errorFailed; manual repair likely required.
StimulusActionState
state.py · drives ActionBar lifecycle
  • idleReady to send. Send Stimulus.
  • submittingPOST in flight. Sending…
  • acceptedServer ack'd. Accepted.
  • measuringResponse window open. Measuring…
  • completedEncounter closed. Send Stimulus.
  • failedRejected or aborted. Retry.
