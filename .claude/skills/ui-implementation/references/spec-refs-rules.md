# Spec-reference rules for UI work

- Preserve existing file/module docstrings that already carry spec grounding.
- When a new UI module needs a file docstring, keep it short and include the governing `§` references when they are known from the plan or source audit.
- Do not invent new `§` references. Use the validated UX plan, current audit, or existing neighboring modules as the source.
- Keep spec grounding descriptive. Do not turn file docstrings into planning documents.
